Teacher-ready lesson • Prisma ORM v7 approach

Connect JavaScript to PostgreSQL or MySQL with Prisma

A complete classroom plan for building a Node.js REST API using Express and Prisma ORM, then connecting a simple HTML/JavaScript frontend to that API.

Learning outcome

By the end, students will be able to:

  • Explain ORM and Prisma architecture.
  • Configure PostgreSQL or MySQL safely.
  • Create models and migrations.
  • Perform Create, Read, Update and Delete operations.
  • Connect an HTML frontend to a Node.js API.

2. Correct Architecture

Important: Prisma must never run directly inside browser HTML or frontend JavaScript. Database credentials would become public. Prisma runs in the Node.js backend. 
HTML + Browser JavaScript
          │
          │ fetch("http://localhost:3000/api/students")
          ▼
Node.js + Express API
          │
          │ Prisma Client
          ▼
PostgreSQL or MySQL Database

4. Select and Configure the Database

PostgreSQL setup

npm install @prisma/adapter-pg pg
npm install @types/pg --save-dev
npx prisma init --datasource-provider postgresql --output ../generated/prisma

Create the database:

CREATE DATABASE prisma_school;

In .env:

DATABASE_URL="postgresql://postgres:YOUR_PASSWORD@localhost:5432/prisma_school?schema=public"

MySQL setup

npm install @prisma/adapter-mariadb
npx prisma init --datasource-provider mysql --output ../generated/prisma

Create the database:

CREATE DATABASE prisma_school;

In .env:

DATABASE_URL="mysql://root:YOUR_PASSWORD@localhost:3306/prisma_school"
DATABASE_HOST="localhost"
DATABASE_PORT="3306"
DATABASE_USER="root"
DATABASE_PASSWORD="YOUR_PASSWORD"
DATABASE_NAME="prisma_school"
Never upload the .env file to GitHub. Add .env to .gitignore.

5. Configure Prisma and Create the Schema

prisma.config.ts

import "dotenv/config";
import { defineConfig, env } from "prisma/config";

export default defineConfig({
  schema: "prisma/schema.prisma",
  migrations: {
    path: "prisma/migrations"
  },
  datasource: {
    url: env("DATABASE_URL")
  }
});

prisma/schema.prisma

Use postgresql for PostgreSQL or mysql for MySQL.

generator client {
  provider = "prisma-client"
  output   = "../generated/prisma"

  provider = "postgresql"

model Student {

  name      String
  email     String   @unique
  course    String
  age       Int?
  createdAt DateTime @default(now())

import "dotenv/config";
import { PrismaPg } from "@prisma/adapter-pg";
import { PrismaClient } from "../generated/prisma/client.js";

const adapter = new PrismaPg({
  connectionString: process.env.DATABASE_URL
});

export const prisma = new PrismaClient({ adapter });
import "dotenv/config";
import { PrismaMariaDb } from "@prisma/adapter-mariadb";
import { PrismaClient } from "../generated/prisma/client.js";

const adapter = new PrismaMariaDb({
  host: process.env.DATABASE_HOST,
  port: Number(process.env.DATABASE_PORT || 3306),
  user: process.env.DATABASE_USER,
  password: process.env.DATABASE_PASSWORD,
  database: process.env.DATABASE_NAME,
  connectionLimit: 5
});

export const prisma = new PrismaClient({ adapter });

src/server.js — complete CRUD API

import express from "express";
import cors from "cors";
import { prisma } from "./prisma.js";

const app = express();
const PORT = process.env.PORT || 3000;

app.use(cors());
app.use(express.json());
app.use(express.static("public"));

// CREATE
app.post("/api/students", async (req, res) => {
  try {
    const { name, email, course, age } = req.body;

    if (!name || !email || !course) {
      return res.status(400).json({
        message: "name, email and course are required"
      });
    }

    const student = await prisma.student.create({
      data: {
        name,
        email,
        course,
        age: age ? Number(age) : null
      }
    });

    res.status(201).json(student);
  } catch (error) {
    console.error(error);
    res.status(500).json({ message: "Unable to create student" });
  }
});

// READ ALL
app.get("/api/students", async (_req, res) => {
  try {
    const students = await prisma.student.findMany({
      orderBy: { createdAt: "desc" }
    });
    res.json(students);
  } catch (error) {
    console.error(error);
    res.status(500).json({ message: "Unable to fetch students" });
  }
});

// READ ONE
app.get("/api/students/:id", async (req, res) => {
  try {
    const student = await prisma.student.findUnique({
      where: { id: Number(req.params.id) }
    });

    if (!student) {
      return res.status(404).json({ message: "Student not found" });
    }

    res.json(student);
  } catch (error) {
    res.status(500).json({ message: "Unable to fetch student" });
  }
});

// UPDATE
app.put("/api/students/:id", async (req, res) => {
  try {
    const { name, email, course, age } = req.body;

    const student = await prisma.student.update({
      where: { id: Number(req.params.id) },
      data: {
        name,
        email,
        course,
        age: age ? Number(age) : null
      }
    });

    res.json(student);
  } catch (error) {
    console.error(error);
    res.status(500).json({ message: "Unable to update student" });
  }
});

// DELETE
app.delete("/api/students/:id", async (req, res) => {
  try {
    await prisma.student.delete({
      where: { id: Number(req.params.id) }
    });
    res.json({ message: "Student deleted successfully" });
  } catch (error) {
    console.error(error);
    res.status(500).json({ message: "Unable to delete student" });
  }
});

app.listen(PORT, () => {
  console.log(`Server running at http://localhost:${PORT}`);
});

process.on("SIGINT", async () => {
  await prisma.$disconnect();
  process.exit(0);
});

7. Create the HTML Frontend

public/index.html

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>Student Manager</title>
  <style>
    body { font-family: Arial; max-width: 850px; margin: 40px auto; padding: 16px; }
    form { display: grid; gap: 10px; margin-bottom: 25px; }
    input, button { padding: 11px; }
    li { margin: 8px 0; padding: 10px; background: #f1f5f9; }
  </style>
</head>
<body>
  <h1>Prisma Student Manager</h1>

  <form id="studentForm">
    <input id="name" placeholder="Student name" required />
    <input id="email" type="email" placeholder="Email" required />
    <input id="course" placeholder="Course" required />
    <input id="age" type="number" placeholder="Age" />
    <button type="submit">Add Student</button>
  </form>

  <h2>Students</h2>
  <ul id="studentList"></ul>

  <script>
    const form = document.getElementById("studentForm");
    const list = document.getElementById("studentList");

    async function loadStudents() {
      const response = await fetch("/api/students");
      const students = await response.json();

      list.innerHTML = students.map(student => `
        <li>
          <strong>${student.name}</strong>
          — ${student.email} — ${student.course}
          <button onclick="deleteStudent(${student.id})">Delete</button>
        </li>
      `).join("");
    }

    form.addEventListener("submit", async event => {
      event.preventDefault();

      const data = {
        name: document.getElementById("name").value,
        email: document.getElementById("email").value,
        course: document.getElementById("course").value,
        age: document.getElementById("age").value
      };

      const response = await fetch("/api/students", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify(data)
      });

      if (!response.ok) {
        alert("Could not add student");
        return;
      }

      form.reset();
      loadStudents();
    });

    async function deleteStudent(id) {
      await fetch(`/api/students/${id}`, { method: "DELETE" });
      loadStudents();
    }

    loadStudents();
  </script>
</body>
</html>

9. Common Errors and Fixes

Error Likely reason Fix
P1000 Wrong username or password. Correct the credentials in .env.
P1001 Database server is unreachable. Start PostgreSQL/MySQL and verify host and port.
P2002 Duplicate value in a unique field. Use a different email or catch the duplicate error.
Missing DATABASE_URL The environment variable was not loaded. Check .env and import "dotenv/config".
Prisma Client not generated Schema changed but client was not regenerated. Run npx prisma generate.
Table does not exist Migration was not applied. Run npx prisma migrate dev.
CORS error Frontend and backend run on different origins. Enable and correctly configure the cors middleware.
